The `IdParserFetcher` interface provides a standard three-step process for fetching identifiers (like DOIs) for bibliographic entries: (1) constructing a URL from a `BibEntry`, (2) parsing the response into a list of `BibEntry` objects using a specified parser, and (3) extracting the identifier from the results. It includes optional post-cleanup of fetched entries and handles errors like malformed URLs or I/O exceptions. The default implementation `findIdentifier` orchestrates these steps and returns an `Optional` identifier.
