The `IdParserFetcher` interface defines a three-step process for fetching identifiers (like DOIs) for bibliographic entries: 1) Construct a URL from a `BibEntry`, 2) Parse the response into a list of `BibEntry` objects using a specified parser, and 3) Extract the identifier from the parsed entries. It includes optional post-cleanup of fetched entries and handles various exceptions like malformed URLs or I/O errors. The default implementation `findIdentifier` combines these steps to return an optional identifier.
